Movies API

Welcome to the Movies API! This API is the backend component of a **movies web application** that allows users to explore information about various **movies, directors, and genres**. Additionally, users can **create accounts**, **update their personal information**, and **manage a list of their favorite movies**.

This document will be updated as development progresses to provide instructions on using the API, including details on routes, HTTP methods, and expected responses.

Current API Status

As of now, the API provides the following basic functionality:

  • GET /: Serves the homepage (index.html).
  • GET /documentation: Serves this documentation page.
  • More features (like user registration and movie management) will be added soon!

API Endpoints

This API exposes a number of endpoints that allow interaction with movie and user data:

1. CREATE a new user

Endpoint URL: /users

Method Type: POST

Description: This endpoint creates a new user. The user will be assigned a unique ID and can later be used for managing their favorite movies.

Request Body Format:

{
  "name": "John Doe"
}

Response Data Format:

{
  "id": "some-unique-id",
  "name": "John Doe"
}

2. UPDATE user details

Endpoint URL: /users/:id

Method Type: PUT

Description: This endpoint updates the name of an existing user based on their unique ID.

Request Body Format:

{
  "name": "Jane Doe"
}

Response Data Format:

{
  "id": "existing-id",
  "name": "Jane Doe"
}

3. CREATE a movie in a user's favorite list

Endpoint URL: /users/:id/movieTitle

Method Type: POST

Description: Adds a movie to the user's list of favorite movies. The movie is specified by the movieTitle parameter.

Request Body Format: None (movie title is passed as a URL parameter)

Response Data Format:

{
  "message": "Movie has been added to user 123's favorite list."
}

4. DELETE a movie from a user's favorite list

Endpoint URL: /users/:id/movieTitle

Method Type: DELETE

Description: Removes a movie from the user's list of favorite movies.

Request Body Format: None (movie title is passed as a URL parameter)

Response Data Format:

{
  "message": "Movie has been removed from user 123's favorite list."
}

5. DELETE a user

Endpoint URL: /users/:id

Method Type: DELETE

Description: Deletes a user from the database based on their unique ID.

Response Data Format:

{
  "message": "User 123 has been deleted."
}

6. GET all movies

Endpoint URL: /movies

Method Type: GET

Description: Returns a list of all available movies in the database.

Response Data Format:

[
  {
    "Title": "Top Gun",
    "Description": "A modern action movie about elite naval aviators.",
    "Genre": {"Name": "Action"},
    "Director": {"Name": "Tony Scott"}
  },
  ...
]

7. GET a movie by title

Endpoint URL: /movies/:title

Method Type: GET

Description: Fetches details of a specific movie based on its title.

Response Data Format:

{
  "Title": "Top Gun",
  "Description": "A modern action movie about elite naval aviators.",
  "Genre": {"Name": "Action"},
  "Director": {"Name": "Tony Scott"}
}

8. GET movies by genre

Endpoint URL: /movies/genre/:genreName

Method Type: GET

Description: Returns all movies belonging to a specified genre.

Response Data Format:

[
  {
    "Title": "Top Gun",
    "Description": "A modern action movie about elite naval aviators.",
    "Genre": {"Name": "Action"},
    "Director": {"Name": "Tony Scott"}
  },
  ...
]

9. GET movies by director

Endpoint URL: /movies/directors/:directorName

Method Type: GET

Description: Fetches all movies directed by a specific director.

Response Data Format:

[
  {
    "Title": "Top Gun",
    "Description": "A modern action movie about elite naval aviators.",
    "Genre": {"Name": "Action"},
    "Director": {"Name": "Tony Scott"}
  },
  ...
]
